2 ЗАВДАННЯ
Основна мета роботи – створити фундаментальну документацію (Vision & Scope) для програмної системи автоматичного догляду за рослинами. Це включає опис ключових елементів: від проблематики та бізнес-логіки до функціональних компонентів.

3. ОПИС ВИКОНАНОЇ РОБОТИ
Було розроблено повну документацію Vision & Scope для програмної системи автоматичного догляду за рослинами, яка включає детальний аналіз бізнес-вимог, технічну концепцію рішення та чітко визначені рамки проєкту. Документація охоплює всі ключові аспекти майбутньої системи: від передумов створення та бізнес-можливостей до технічної архітектури з використанням сучасного технологічного стеку.
У процесі роботи було проведено аналіз існуючих рішень на ринку (Xiaomi Mi Flora, Parrot Pot, Click & Grow Smart Garden), виявлено їх основні недоліки та визначено конкурентні переваги планованої системи. Розроблено концепцію комплексного IoT-рішення з автоматичним моніторингом параметрів навколишнього середовища та інтелектуальним прийняттям рішень щодо догляду за рослинами.
Деталізовано архітектуру системи, яка включає серверну частину на Node.js з Express.js та TypeScript, IoT-клієнт на базі Arduino з WiFi-модулем, веб-інтерфейс на React.js та мобільний додаток на React Native. Визначено функціональні можливості для первинного випуску та подальших ітерацій розвитку продукту. Опис проєкту у форматі Vision & Scope наведено у додатку Б.

4. ВИСНОВКИ
Розроблено концептуальний документ Vision & Scope, який чітко визначає бізнес-рамки та технічне бачення системи автоматичного догляду за рослинами. У процесі роботи набуто цінні навички аналізу бізнес-вимог, планування архітектури IoT-систем та формування технічних специфікацій для інтеграції різнорідних компонентів у єдину екосистему. Розроблений документ створює надійну основу для подальшого етапу проектування та розробки, забезпечуючи чітке розуміння цілей проєкту, критеріїв успіху та можливість ефективного планування ресурсів для реалізації кожного компоненту системи.

ДОДАТОК Б

Концепція та рамки проєкту (PROJECT VISION AND SCOPE)

1.	BUSINESS REQUIREMENTS (БІЗНЕС-ВИМОГИ)
1.1.	Background (Передумови) 
Система вирішує проблему неефективного догляду за кімнатними рослинами, яка призводить до їх загибелі або поганого розвитку. За статистикою, більшість власників кімнатних рослин стикаються з проблемами неправильного поливу, недостатнього освітлення або несвоєчасного реагування на зміни умов навколишнього середовища.
Поточні підходи базуються на візуальному огляді та інтуїції, що часто призводить до переливу або недостатнього поливу. Відсутність точного моніторингу параметрів середовища (вологість ґрунту, температура, освітленість) не дозволяє забезпечити оптимальні умови для росту різних видів рослин, особливо під час відпусток або тривалої відсутності власника.
1.2.	Business Opportunity (Бізнес-можливості)
На ринку існують такі рішення як Xiaomi Mi Flora (лише моніторинг без автоматизації), Parrot Pot (висока вартість, обмежена функціональність), Click & Grow Smart Garden (працює тільки з власними картриджами), PlantLink (припинив роботу через технічні проблеми). Більшість систем фокусуються виключно на поливі та не враховують комплексний підхід до догляду.
Основні недоліки існуючих рішень: висока вартість, обмежена кількість підтримуваних рослин, складність. Нова система покращить ситуацію через доступну вартість, універсальність для різних типів рослин та інтуїтивний інтерфейс.

2.	VISION OF THE SOLUTION (КОНЦЕПЦІЯ ТА РАМКИ ПРОЄКТУ)КОНЦЕПЦІЯ РІШЕННЯ)
2.1.	Vision Statement (Окреслення концепції)
Система забезпечує автоматичний комплексний догляд за кімнатними рослинами через мережу IoT-датчиків та виконавчих механізмів. Вона відстежує ключові параметри (вологість ґрунту, температуру, освітленість) та автоматично підтримує оптимальні умови для кожного виду рослини.
Система створює цінність через збереження здоров'я рослин, економію часу користувача (до 90% зменшення ручного догляду), зменшення споживання води (до 30% через точковий полив) та навчання користувачів правильному догляду через персоналізовані рекомендації та сповіщення.
2.2.	Major Features (Головна функціональність)
Система працює за принципом замкнутого циклу: датчики збирають дані кожні 15 хвилин, алгоритм порівнює їх з налаштованими параметрами для конкретного виду рослини, приймає рішення про необхідні дії та активує відповідні механізми (полив, увімкнення LED-освітлення, активацію вентилятора).
Користувачі взаємодіють з системою через мобільний додаток: обирають тип рослини з бази даних, отримують push-сповіщення про критичні стани, переглядають графіки змін параметрів за тиждень/місяць, налаштовують режими (відпустка, зима/літо) та отримують персоналізовані поради з догляду на основі зібраної раніше інформації.

3.	SCOPE AND LIMITATIONS (КОНЦЕПЦІЯ ТА РАМКИ ПРОЄКТУ)РАМКИ ТА ОБМЕЖЕННЯ ПРОЄКТУ)
3.1.	Scope of Initial Release (Рамки первинного випуску)
Серверна частина включає розробку бази даних, реалізацію REST API з CRUD операціями для управління користувачами, рослинами, датчиками та історією вимірювань, бізнес логіку для обробки даних з IoT пристроїв та прийняття рішень про автоматичні дії, функції адміністрування (управління користувачами, моніторинг системи, налаштування параметрів), та систему аутентифікації з ролями доступу. API підтримує ендпоінти для реєстрації/авторизації користувачів, CRUD операції з профілями рослин, отримання телеметрії та історії даних, управління налаштуваннями автоматизації.
IoT-клієнт на базі Arduino з WiFi модулем реалізує збір даних з датчиків (вологість ґрунту через ємнісний датчик, температура/вологість повітря, освітленість через фоторезистор), передачу телеметрії на сервер через HTTP/MQTT протоколи кожні 15 хвилин, управління виконавчими механізмами, функції налаштування (конфігурація WiFi, зміна інтервалів відправки даних).
3.2.	Scope of Subsequent Releases (Рамки наступних випусків)
Веб-інтерфейс надає розширену аналітику з графіками динаміки параметрів за різні періоди, експорт даних у CSV/PDF формати, налаштування системи з можливістю створення власних профілів рослин, та адміністративну панель для налаштування алгоритмів автоматизації. Інтерфейс також включає календар подій догляду та систему нагадувань для ручних операцій.
Мобільний додаток (Android) забезпечує швидкий доступ до поточного стану всіх рослин, push-сповіщення про критичні події (низька вологість, висока температура), можливість ручного керування функціями (примусовий полив, тест освітлення). Обидва клієнти взаємодіють з сервером через захищений HTTPS REST API з JWT автентифікацією та мають локалізацію (англійська/українська мови).

4.	Business Context (БІЗНЕС-КОНТЕКСТ)
4.3.	Operating Environment (Робоче середовище)
Серверна частина розробляється на Node.js з Express.js фреймворком та TypeScript для типізації, використовуючи PostgreSQL з ORM Sequelize для роботи з базою даних, MQTT брокер Mosquitto для комунікації з IoT пристроями, та JWT для аутентифікації користувачів. API документується через Swagger/OpenAPI, а для валідації запитів використовується Joi. Система управління версіями - Git з GitHub репозиторієм, CI/CD налаштовується через GitHub Actions з автоматичним тестуванням та деплоєм.
IoT-клієнт програмується на Arduino IDE з мовою C++, використовуючи бібліотеки WiFiManager для налаштування мережі, PubSubClient для MQTT комунікації, ArduinoJson для обробки JSON повідомлень, DHT sensor library для роботи з датчиками температури/вологості. 
Веб-клієнт створюється на React.js з TypeScript, Material-UI для компонентів інтерфейсу, Recharts для візуалізації даних, Redux для управління станом, та Axios для HTTP запитів. 
Мобільний додаток розробляється на React Native з Expo SDK, використовуючи React Navigation для навігації та AsyncStorage для локального зберігання. Контейнеризація здійснюється через Docker з docker-compose для локальної розробки.
